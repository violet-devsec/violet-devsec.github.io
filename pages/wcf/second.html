CREATING A WCF SERVICE ACCESSIBLE VIA HTTPS

October 14, 2020 Violet Devsec
  1. Add both files to the client application.

  2. Go to the run window and type "mmc", hit enter. Then in the window that opens (called the "Microsoft Management Console, for those who care) perform the following actions. File -> Add/Remove Snap-in -> Add… -> Double click Certificates -> Select Computer Account and Click Next -> Finish -> Close -> OK

  3. Create the Root CA cert in host machine using “makecert”.
  4. Move the Root CA cert to Trusted Root Node from Personal Node by drag and drop.
  5. Use “makecert” again to create Server certificate.

    *-n should be the host machine’s name.

  6. Export the CA certificate to .cer format.
    7. Exchange the .cer file to the client machine and install the exported certificate.
  7. Add it to Trusted Root node.
  8. Bind the SSL cert to the application port in the Host machine.

    (Start administrator powershell to perform this operation) Certhash=thumbprint of ssl certificate, appid=Host application’s assembly info Id

  9. Start the host service in the host machine
  10. Start the client app in the client machine and start using the service using https.
